════════════════════════════════════════
CODE REVIEW CHECKLIST
════════════════════════════════════════
C/eBPF:
- Byte order (host vs network endian)
- BPF verifier constraints (unbounded loops, stack size, helper availability)
- CO-RE correctness (BPF_CORE_READ vs direct dereference)
- Missing NULL checks and map lookup error handling
- Kernel version requirements for helpers or map types

Go:
- Struct alignment when used as BPF map keys/values (must match C layout)
- Endianness when reading map data from kernel (encoding/binary)
- Goroutine leaks, missing context cancellation, unclosed map/program FDs
- Improper use of unsafe.Pointer or uintptr
- Missing error wrapping (fmt.Errorf with %w) and silent error discards
- Ringbuf/perf buffer consumer goroutine lifecycle management

React/TypeScript:
- Stale closures in useEffect, useCallback, useMemo
- Missing or incorrect dependency arrays
- Unnecessary re-renders (missing memo boundaries)
- Type assertions (as X) hiding real type errors
- WebSocket/SSE cleanup on unmount
- Large bundle imports (import entire recharts vs tree-shaking)
- Unhandled promise rejections in event handlers

Design:
- Contrast ratio (WCAG AA minimum, AAA for critical data)
- Visual hierarchy — is the most important data immediately obvious?
- Color used as the only differentiator (colorblind accessibility)
- Overloaded charts — too many series, missing baselines, misleading axes
- Inconsistent spacing/padding breaking visual rhythm
- Loading and empty states — never leave the user staring at nothing
